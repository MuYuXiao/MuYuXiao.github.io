广度优先搜索与深度优先搜索

2021-03-05

首先引入图的概念
如上图我们把每一个元素作为一个点，称这个点为“节点”；有关的节点用线连起来，称这些线段“边”
(图为无向图)

通俗来说，深度优先即为一条路走到黑，然后再走下一条路，而广度优先则是先到下一个路口
即深度优先每次到下一级，直至无子节点是再从最高级未搜索结点开始；广度优先则先探索下一级所有子 节点，之后在搜索下一级节点

假设指针在A处开始搜索，对于广度优先来说搜索队列为A B G F C I H E D
对于深度优先则是A B C D I G H E F

关于实现，简单来说就是构建一个搜索队列，将要搜索的节点输入队列里。搜索即可。

const int TREE_SIZE = 9; 
std::stack<node*> visited, unvisited; 
node nodes[TREE_SIZE]; 
node* current; 
for( int i=0; i<TREE_SIZE; i++) //初始化树 
    {    
        nodes[i].self = i;   
        int child = i*2+1;    
        if( child<TREE_SIZE ) //Left child       
        nodes[i].left = &nodes[child];    
        else nodes[i].left = NULL;    
        child++;    
        if( child<TREE_SIZE ) //Right child           
        nodes[i].right = &nodes[child];    
        else       nodes[i].right = NULL;
    }             
unvisited.push(&nodes[0]); //先把0放入UNVISITED stack  
while(!unvisited.empty()) //只有UNVISITED不空
